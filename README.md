# 📊 AIBA — 次世代技術投資分析システム

**Advanced Investment & Behavior Analytics**

世間の熱狂による「割高な高値掴み」を避け、水面下で熱を帯びている次世代技術領域の最適な投資タイミング（押し目・初動）を定量的に検知する。テクニカル指標とセンチメント指標の**乖離**を分析し、データドリブンな投資判断を支援するダッシュボード。

- **本番稼働中**：GitHub Actions が平日 日次でデータ収集・スコア計算・予測 → Supabase 蓄積 → Vercel で公開。
- セットアップ・運用手順は **[SETUP.md](./SETUP.md)** を参照。

## ✨ 主な機能

### 買い（発掘）
- **AIBAスコア v1.2**：層別重み付け＋RSIペナルティの総合「買い時度」(0–100)
- **成長×割安スコア**／**センチメント傾き(↑↓)**／**🔀乖離**（センチメント先行×株価出遅れ＝仕込み好機）
- **保有目安バッジ（買いの"型"）**：🌱成長型（構造的成長・長期向き）／⚡反発型（売られすぎリバウンド・短期向き）／⚖️両取り
- **1ヶ月先予測**：買い場入り確率＋予測AIBA（平均回帰＋ロジスティック。`predict.py --backtest` で naive 比較検証済み）
- **購入目安**：妥当値(25日MA)・押し目買い目安(−1σ)・下値支持(60日安値)を銘柄詳細に表示
- **下方リスク・下値メド**：52週安値（下落余地%）＋近い支持(−2σ/60日安値)、年率ボラ・1年最大下落率から**値動きの安定度（下方耐性）**を提示
- **⭐ Pickupページ**：地域・ETF/個別を問わず「今買い」候補を横断抽出（**前日順位との変動 ▲▼/NEW** を併記）
- **🔎 スクリーナー** `/screener`：地域/種別/階層/AIBA/買い場確率/予想PER/熱量/乖離の多条件を即時フィルタ＋並び替え。**条件は localStorage に自動保存**

### 売り（保有管理）
- **マイ・ポートフォリオ**（ログイン・アカウント紐付け）：保有銘柄を追加/編集/削除。**投信**は同じ指数を追う代用ETFで評価
- **過熱度**＋**売り場ハイライト**＋**売りシグナル合成**（テクニカル過熱＋ファンダ〈高PER/減益〉＋決算接近）
- **損切りライン**（任意・%可変）：取得単価から設定％下落で「損切り検討」
- **保有期間別の判定**（短期〜1ヶ月／中期1〜6ヶ月／長期半年〜）：期間ごとに適した指標群で「売り / 継続保有」を集計表示
- 平均取得単価比の**損益％**、次回決算日アラート
- **配分分析**：保有のテーマ別/地域別エクスポージャー＋集中度（最大比率・実効銘柄数＝1/HHI）
- **今後の決算カレンダー**：保有銘柄の決算予定日を日付昇順でカード表示（7日以内=赤・30日以内=橙）

### 銘柄詳細
- **一行ナラティブ**（自動要約）：AIBA＋相対PER＋業績＋熱量＋予測＋決算接近を1〜2文に
- **健康度レーダー**：AIBA/割安/熱量/非過熱/押し目度/**事業の頑丈さ**＋**業界平均を重ねて比較**
- **事業の頑丈さ（0–100）**：営業利益率・ROE・D/E・流動比率・FCF の合成スコア
- **相対PERフェアバリュー**：同地域・同テーマのピア中央値比で「◯%割安/割高」
- **時系列チャート**：株価×スコア二軸、買い場/売り場ハイライト、ボリンジャーバンド、MACD、通貨対応、期間切替
- **テクニカル総合判定**：移動平均(25/200)・MACD・一目均衡表＋RSI・ボリンジャー・ストキャスを買い/売り/中立で網羅

### 横断・補助
- **地域タブ**（Global / 米国 / 日本 / その他）× **業界ETF / 個別株トグル**
- **🧭 テーマ一覧** `/themes`：全テーマを階層別に俯瞰（研究熱量＋業界AIBA＋ニュース論調〈GDELT平均トーン〉）＋**🌱 新興テーマ候補**を熱量実測で動的提示

## 🧮 AIBAスコア v1.2

階層に応じて指標の重みを動的に変える 0–100 のスコア（[`backend/aiba/score.py`](./backend/aiba/score.py)）。

**インプット**
- **テクニカル（事実）**：株価・出来高・RSI(14)・移動平均(25)乖離率（`yfinance`）
- **センチメント（先行）**：GitHub（新規リポジトリ＋コミット活動量）× arXiv（論文）× Hacker News × Google Trends × 特許（EPO OPS）× ニュース報道量（GDELT）の加重平均増加率（本体=GitHub/arXiv、補助=HN/Trends/特許/ニュース。最低ボリューム/信号数ガードあり）

**ロジック**
- テクニカル割安感＝RSI割安スコアと移動平均乖離スコアの平均
- 層別重み＝第1層 (T0.4/S0.6)・第2層 (0.5/0.5)・第3層 (0.3/0.7)
- RSIが50超で過熱ペナルティ → 重み付き合算 − ペナルティ（0–100にクランプ）

## 🏗️ アーキテクチャ

運用コストを無料枠に抑えた構成。**重い処理は GitHub Actions、Vercel は表示のみ**。  
表示は **ISR（10分キャッシュ）＋ 重い集計の `unstable_cache`** で高速化。長いランキングは**初期表示＋「もっと見る」**で初期描画を軽量化。

```
yfinance / GitHub / arXiv / Hacker News / Google Trends / EPO OPS（特許）/ GDELT（ニュース）
        │ 収集・スコア計算・予測（Python）
        ▼  ── GitHub Actions（日次cron＋週次cron）
Supabase (PostgreSQL Free)
        │ 公開データは anon キーで読み取り / RLS 保護
        │ ユーザーデータ(ウォッチリスト/保有)は本人のみ
        ▼
Next.js ダッシュボード (Vercel)
```

### ディレクトリ構成
```
AIBA-project/
├── config/targets.yaml          # 監視ユニバース（16テーマ×地域×ETF/個別株）＋新興テーマ候補
├── db/                          # Supabase スキーマ
│   ├── schema.sql               #   domains / daily_metrics / RLS
│   ├── predictions.sql          #   予測

│   ├── user_portfolio.sql       #   user_holdings / ticker_metrics / ticker_fundamentals
│   ├── backtest.sql             #   バックテスト結果
│   ├── snapshots.sql            #   スコア定点記録（out-of-sample 評価）
│   ├── candidates.sql           #   新興テーマ候補の熱量
│   └── benchmark.sql            #   ベンチマーク指数の終値（ACWI/QQQ/ARKK/BUZZ）
├── backend/                     # データ収集・スコア・予測（Python）
│   ├── aiba/{config,technical,sentiment,score,forecast,db,pipeline}.py
│   ├── run_daily.py             #   日次スコア
│   ├── predict.py               #   1ヶ月先予測（--backtest で検証）
│   ├── portfolio_job.py         #   保有ティッカーの過熱度・ファンダ
│   ├── fundamentals_job.py      #   ユニバース個別株のファンダ
│   ├── candidates_job.py        #   新興テーマ候補の熱量（週次）
│   ├── benchmark_job.py         #   ベンチマーク指数（ACWI 等）の終値取得
│   ├── backtest.py              #   IC・買い優位の算出（--save）
│   ├── snapshot.py              #   定点記録の記録＋評価（--backfill）
```

日次ジョブの流れ：`run_daily → predict → portfolio_job → fundamentals_job → benchmark → backtest → snapshot → check_data → notify`（timeout 45分）。  
週次ジョブ：`candidates_job`（新興テーマ候補の熱量）／`ic_monthly_job`（月次IC）／`news_tone_job`（テーマ別ニュース論調＝GDELT平均トーン）。

### セキュリティ：パスワード保護

Supabase の組み込み機能は Pro 以上限定のため、**Free 枠ではクライアント側で同等の保護を自前実装**（[`frontend/lib/password.ts`](./frontend/lib/password.ts)）。

- **漏洩チェック**：新規登録時に **HaveIBeenPwned Pwned Passwords range API**（k-匿名性・SHA-1先頭5桁のみ送信）で照合し、漏洩済みパスワードを拒否。
- **強度ポリシー**：10文字以上＋（英大/英小/数字/記号の）3種類以上を必須化。
- ネットワーク失敗時はフェイルオープン（登録を妨げない）。

## 📈 キャリブレーション記録

out-of-sample で定点観測し、重み調整の判断材料を残す（`backend/backtest.py` / `snapshot.py`、H=21営業日基準）。詳細は `/verify`（検証）で常時確認可能。

### 主な結論（2026-06・2022〜2026で再評価）
- **センチメントが選別力の源泉**：クロスセクションICで、センチメント **+0.037（IR +0.25・最も安定して正）**。一方 **AIBA −0.019 / テクニカル −0.025 と僅かに負**＝この**モメンタム主導相場では「割安・売られすぎ」の逆張り成分が相対的に劣後**したため。第1層は**純センチメント(w_tech=0)でIC +0.148**と強い。
  - ※ICの絶対値が小さいのは正常（市場は概ね効率的。単一factorは0.02〜0.05でも有意）。テクニカルの負ICは**地合い依存**（調整・レンジ相場では正に転じ得る）。
- **買い優位は短期で安定**：レジーム拡張（54アンカー）で 1ヶ月 **+2.8pt（勝率61%）**。3〜6ヶ月は小幅、12ヶ月はリターンほぼ互角。`AIBA≥60` 買いコホートは平均 +3.73%（全体 +2.65%）。
- **最適重みはセンチメント寄り**：層別 最適 w(テク) はいずれも 0.0。現行 L1 0.7（テクニカル重視）は**見直し候補**だが、2022-2024のセンチメント中立バイアスもあり**様子見**。
- **疑似エクイティカーブ**（月次リバランス・100→・2022〜）：**≥60 = 443** が分散の効いた頑健な優位（ユニバース等ウェイト 389・全世界株ACWI放置 168 を上回る）。≥70 は488だが高分散・低再現性。

**先行性の現状評価**：センチメントの**正のクロスセクションIC＋IR**はこの仮説を**支持する兆し**だが、(1)実センチメント履歴が実質1.5年と短い、(2)ほぼ単一の上昇レジーム、(3)生存者バイアス＋in-sample のため**"有望な仮説"段階＝実証はこれから**。ICは `/verify` に**時系列グラフ**化し、時間経過で先行性が持続するかを評価できるようにしている。

**重要な注意（過大評価バイアス）**：ユニバースは"今の勝ち組"を後知恵で選定（生存者バイアス）、2022-2024は**センチメント中立で再構築**、月次アンカーは**先行期間が重複**。絶対リターンでなく**相対優位・頑健性**として読むこと（取引コスト未考慮の概算）。

> 2026-06 保守メモ：センチメント統合に**単一ソース/低ボリュームで極値が刻まれるバグ**があったため、**加重ブレンド＋最低信号数・最低ボリュームのガード**を導入して修正済み。さらに**ニュース報道量（GDELT）**を補助信号に追加。上記ICはこれら変更前データでの集計のため、今後の蓄積で再評価する。

### v1.2 重み改定（2026-06）
**データ監査**：DBのセンチメントを月次で確認すると、**2022-2024は全行ぴったり50.0（中立再構築＝本物でない）**、**本物のセンチメントは2025-01以降の約18ヶ月のみ**と判明。よって先行性ICの評価に使えるのは実質2025+。

**再計算（2025+のクロスセクションIC）**：第1層は **純センチメントIC +0.146（システム最強）／純テクニカル 負**。現行 v1.1 は **テクニカル0.7** と逆だった。

**改定**：第1層を **テク0.7/センチ0.3 → 0.4/0.6**（センチメント寄りの中間）。第2/3層は据置。**過去スコアは v1.1 のまま**（定点記録の point-in-time 性を保持）。

## 🔬 派生スコアのロジックと監査

銘柄詳細・ポートフォリオ等で使う派生スコアの定義・しきい値・近似/前提。`/guide`（利用者向け説明）に対し、ここは**式レベルの仕様**。

| スコア（実装） | 定義（要点） | 主なしきい値・根拠 | 近似/前提 |
|---|---|---|---|
| **事業の頑丈さ** 0–100<br>`fundamentals.ts:qualityScore` | 収益性(営業利益率・ROE)／財務健全性(D/E・流動比率)／キャッシュ創出(FCF符号) の**利用可能指標の平均** | 営業利益率 20%≈70点/40%で満点・赤字で30以下／ROE 15%≈76点／D/E 100%≈55点・222%で0／流動比率 2.0≈90点／FCFプラス85・マイナス25 | D/E は yfinance の%表記前提。指標2つ未満は非表示 |
| **下方リスク・下値メド**<br>`indicators.ts:downsideProfile` | 下値メド=52週安値／近い支持=max(60日安値, −2σ)／安定度=年率ボラ＆1年最大ドローダウン | 安定度: ボラ<35%&DD>−35%→高 / <60%&>−55%→中 / 他→低 | テクニカルな目安で割れることあり |
| **保有期間別 売り/継続**<br>`indicators.ts:holdingHorizons` | 期間別に適した指標群の**売り票 vs 継続票**で判定 | 短期=過熱/RSI/ストキャス/BB/MACD・中期=MACD/一目/MA25,75・長期=MA200/52週/雲/熱量 | 票が拮抗は中立 |
| **テクニカル総合判定**<br>`indicators.ts:technicalSummary` | トレンド系(MA25/200・MACD・一目)＋オシレーター系(RSI・BB・ストキャス)を 買い/売り/中立 で集計 | 総合: 買い−売り の差で買い寄り〜売り寄り | 一目・ストキャスは**終値ベース近似** |

AIが「AIはメモリで動く」時代に入ったことを示す研究・開発活動（GitHub/arXiv）は2024年前半から急増していたが、`advanced_semiconductor` のキーワードが **compute 層（cuda / gpu kernel / tensorrt）に偏っており memory/storage 層を捉えていなかった**。

| 見逃した先行シグナル | 発生時期 |
