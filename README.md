# 📊 AIBA — 次世代技術投資分析システム

**Advanced Investment & Behavior Analytics**

世間の熱狂による「割高な高値掴み」を避け、水面下で熱を帯びている次世代技術領域の最適な投資タイミング（押し目・初動）を定量的に検知する。テクニカル指標とセンチメント指標の**乖離**を分析し、データドリブンな投資判断を支援するダッシュボード。

- **本番稼働中**：GitHub Actions が平日 日次でデータ収集・スコア計算・予測 → Supabase 蓄積 → Vercel で公開。
- セットアップ・運用手順は **[SETUP.md](./SETUP.md)** を参照。

## ✨ 主な機能

### 買い（発掘）
- **AIBAスコア v1.1**：層別重み付け＋RSIペナルティの総合「買い時度」(0–100)
- **成長×割安スコア**／**センチメント傾き(↑↓)**／**🔀乖離**（センチメント先行×株価出遅れ＝仕込み好機）
- **保有目安バッジ**：🌱長期（構造的成長）／⚡短期（リバウンド）／⚖️両面
- **1ヶ月先予測**：買い場入り確率＋予測AIBA（平均回帰＋ロジスティック。`predict.py --backtest` で naive 比較検証済み）
- **購入目安**：妥当値(25日MA)・押し目買い目安(−1σ)・下値支持(60日安値)を銘柄詳細に表示
- **⭐ Pickupページ**：地域・ETF/個別を問わず「今買い」候補を横断抽出
- **🔎 スクリーナー** `/screener`：地域/種別/階層/AIBA/買い場確率/予想PER/熱量/乖離の多条件を即時フィルタ＋並び替え

### 売り（保有管理）
- **マイ・ポートフォリオ**（ログイン・アカウント紐付け）：保有銘柄を追加/編集/削除
- **過熱度**＋**売り場ハイライト**＋**売りシグナル合成**（テクニカル過熱＋ファンダ〈高PER/減益〉＋決算接近）
- 平均取得単価比の**損益％**、次回決算日アラート
- **配分分析**：保有のテーマ別/地域別エクスポージャー＋集中度（最大比率・実効銘柄数＝1/HHI）

### 銘柄詳細の分析
- **一行ナラティブ**（自動要約）：AIBA＋相対PER＋業績＋熱量＋予測＋決算接近を1〜2文に
- **健康度レーダー**：AIBA/割安(テク)/熱量/非過熱/押し目度/業績を可視化
- **相対PERフェアバリュー**：同地域・同テーマのピア中央値比で「◯%割安/割高」
- **時系列チャート**：株価×スコア二軸、買い場/売り場ハイライト、**ボリンジャーバンド**、**MACD**、通貨対応（円/ドル）、期間切替（1M〜全期間）

## 🧮 AIBAスコア v1.1

階層に応じて指標の重みを動的に変える 0–100 のスコア（[`backend/aiba/score.py`](./backend/aiba/score.py)）。

**インプット**
- **テクニカル（事実）**：株価・出来高・RSI(14)・移動平均(25)乖離率（`yfinance`）
- **センチメント（先行）**：GitHub（新規リポジトリ＋**コミット活動量**）× arXiv（論文）× **Hacker News** × **Google Trends** × **特許**（EPO OPS・公開件数）の平均増加率（取得できた指標のみ平均）

**ロジック**
- テクニカル割安感＝RSI割安スコアと移動平均乖離スコアの平均
- 層別重み＝第1層 (T0.7/S0.3)・第2層 (0.5/0.5)・第3層 (0.3/0.7)
- RSIが50超で過熱ペナルティ → 重み付き合算 − ペナルティ（0–100にクランプ）

## 🏗️ アーキテクチャ

運用コストを無料枠に抑えた構成。**重い処理は GitHub Actions、Vercel は表示のみ**。

```
yfinance / GitHub / arXiv / Hacker News / Google Trends / EPO OPS（特許）
        │ 収集・スコア計算・予測（Python）
```

## セキュリティ：パスワード保護

Supabase の lint「Leaked Password Protection Disabled」への対応。
組み込み機能は Pro 以上限定のため、**Free 枠ではクライアント側で同等の保護を自前実装**（[`frontend/lib/password.ts`](./frontend/lib/password.ts)）。

- **漏洩チェック**：新規登録時に **HaveIBeenPwned Pwned Passwords range API**（k-匿名性・SHA-1先頭5桁のみ送信、本体は送らない・APIキー不要）で照合し、漏洩済みパスワードを拒否。
- **強度ポリシー**：10文字以上＋（英大/英小/数字/記号の）3種類以上を必須化。
- ネットワーク失敗時はフェイルオープン（登録を妨げない）。
- 将来 Pro へ上げた場合は Supabase Auth 設定で公式機能を有効化すれば二重で担保。

## 📈 キャリブレーション記録

スコアの妥当性を定点観測し、重み調整の判断材料を残す（`backend/backtest.py`、H=21営業日先リターン基準）。

### 2026-06（履歴拡張：2025年通年をバックフィル後・サンプル29,298）
- **IC**：AIBA −0.008 / テクニカル −0.006 / センチメント −0.008（いずれもほぼ0）
- **買い戦略**（AIBA≥60）：平均1ヶ月先 **+4.58%**（全体 +4.30% ＝ +0.28pt優位）対象3,745件
- **層別 最適 w(テクニカル)**：第1層 0.0 / 第2層 0.5 / 第3層 1.0
- **重要な注意（解釈）**：2025年の拡張分は**センチメントを30日間隔で再構築し前方補完**したため、直近6ヶ月の週次データに比べ**センチメント信号が粗く（階段状）、ICが構造的に過小評価**される。第3層のw(テク)=1.0 もこの粗さの副作用とみられ、**現行重みを下げる根拠にはしない**。
- **判断**：長期×粗いセンチメントでは買い優位が薄まる一方、**直近・週次の高解像度窓では優位が明確**（下記6ヶ月窓）。重みは引き続き**現状維持**。拡張データは主に**テクニカル・価格のバックテスト深度**に活用し、センチメント評価は週次窓を基準とする。

### 2026-06（履歴6ヶ月・センチメント週次バックフィル）
- **IC**（先行リターンとの順位相関）：AIBA +0.053 / テクニカル −0.003 / **センチメント +0.130**
- **買い戦略**（AIBA≥60）：平均1ヶ月先 **+8.05%**（全体 +5.38% ＝ +2.7pt優位）
- **層別 最適 w(テクニカル)**：第1層 0.0（IC 0.248）/ 第2層 0.4（0.081）/ 第3層 0.0（0.147）
- **示唆**：現行重み（L1 0.7 / L2 0.5 / L3 0.3）は**センチメントを過小評価**の可能性。週次化でセンチメント先行性が顕著に改善（月次→週次：IC 0.022→0.130、買い優位 +0.9pt→+2.7pt）。
- **判断**：履歴が浅く先行リターンも重複するため**現状維持（様子見）**。候補の穏当な再調整＝L1 0.5/0.5・L2 0.4/0.6・L3 0.25/0.75。**次回（3ヶ月時点）に再評価**。

### 2026-06 定点記録（out-of-sample・2025年通年の月次スナップショット18回で評価）
2025年全体をバックフィルし、月次アンカーで全ホライズンを評価（旧版は直近7回のみで6ヶ月が空欄だった）。
- **1ヶ月**：買い判定(AIBA≥60) +3.50%（全体 +3.30% / 勝率54% / n=213）→ **ほぼ互角**（+0.2pt）
- **3ヶ月**：買い判定 **+19.83%**（全体 +14.58% / 勝率66% / n=171）→ **優位 +5.3pt**
- **6ヶ月**：買い判定 **+52.20%**（全体 +33.72% / 勝率82% / n=142）→ **優位 +18.5pt**
- **示唆**：通年サンプルでは**中期（3〜6ヶ月）ほど買い優位が拡大**。直近窓のみで見えた「1ヶ月特化・3ヶ月劣後」は期間バイアスだった可能性。
- ※注意：2025年は強い上昇相場で**絶対リターンは高め**、AIBAは2025分が**粗い30日センチメント**で算出、月次アンカーは**先行期間が重複**（独立試行でない）。相対優位（買い vs 全体）として読むこと。`/verify`（📊検証）で常時更新・確認可能。

1. センチメント指標の精度向上と拡張

　セクター別の指標重み付け（可変ウェイト）の導入
すべてのテーマに対して一律の計算式を適用するのではなく、セクターごとにGitHubやarXivのデータが持つ「株価への先行性」を評価し、スコアの重み付けを最適化します。生成AIやバイオインフォマティクスには高いウェイトを置き、インフラやエネルギー領域には別のウェイトを適用することで、スコアの信頼性が向上します。

　自然言語処理（NLP）を用いた質的センチメント評価
単純な論文数やコミット数のカウント（量）に加えて、テキストデータの「質」を評価に組み込みます。Transformerモデルなどを用いて論文のアブストラクトやリポジトリの更新内容を解析し、「画期的な技術的進歩」を含むものに高いスコアを付与することで、より精度の高い先行指標となります。

　オルタナティブデータ（代替データ）の追加
研究熱量だけでは測りにくいテーマ（冷却・電力インフラなど）を補完するため、特許の出願動向や、決算説明会（アーニングスコール）での特定キーワードの出現頻度などを新たなセンチメントデータとして組み込むと、網羅性が飛躍的に高まります。

2. 相場環境（レジーム）への適応力強化

　トレンド判定によるRSIペナルティの動的調整
現在の「RSI 50超で一律減点」というルールは、強い上昇トレンド相場では機会損失を生みます。例えば、移動平均線の傾きやADX（平均方向性指数）を用いて「現在はレンジ相場か、トレンド相場か」をシステムに判定させ、強いトレンドが発生している場合は、ペナルティの基準値をRSI 60や70に引き上げる「動的閾値」の導入が有効です。

　「逆張り（押し目）」と「順張り（モメンタム）」のスコア分離
現在のAIBAスコア（押し目買い特化）に加え、あえてトレンドに乗るための「モメンタムスコア」を並行して提示するアプローチです。ユーザーが自身の投資スタイル（安全に拾いたいか、勢いに乗りたいか）に合わせて指標を使い分けられるようになります。

3. ユーザーの納得感・信頼性の向上

　過去データのバックテスト結果の可視化
「AIBAスコアが〇〇以上の時に買っていれば、1ヶ月後・半年後にどうなっていたか」という過去の検証（バックテスト）データを、チャート上や別タブでユーザーが確認できるようにします。これにより、システムの優位性が事実として証明され、ユーザーが自信を持ってツールを利用できるようになります。


私：短期保有も考慮してますが、本業が優先で投資はできるだけ放置・長期保有をターゲットにしたサービスです。
↓
とある方：
　本業優先・放置の方針とズレている点（懸念点）
テクニカル指標のタイムスパンが短すぎる:
現在のロジックでは「25日移動平均線からの乖離」や「14日間のRSI」が採用されています。これらは数週間〜1ヶ月程度のスイングトレードで押し目を拾うための指標です。
「1ヶ月の買い場確率」が監視を要求する:
「あと少しで買い場が来るかもしれない」という確率が提示されると、ユーザーはどうしても高頻度でダッシュボードを確認したくなります。これは「できるだけ放置したい」という目的に反し、本業の集中を削ぐノイズになりかねません。
「放置・長期保有」に最適化するための改善アイデア
本業を優先しつつ、この優れたセンチメント分析を活かすためには、システム側で「ノイズを減らし、必要な時だけ行動を促す」設計にシフトさせるのが効果的です。
長期指標へのアルゴリズム変更:
「25日移動平均」を「200日移動平均」などの長期トレンドラインに変更し、日々のRSIの変動によるペナルティを緩和することで、数ヶ月に1度だけ発生する「真の長期的な買い場」だけを抽出するようにロジックを調整します。
プッシュ通知による「完全放置」の実現:
ユーザーが自らサイトに見に行くのではなく、過去のアプリケーション開発などでも構築されていたSlack通知やメール通知の仕組みを活用し、「AIBAスコアが〇〇を超えた（あるいは長期の乖離が発生した）時だけアラートを飛ばす」機能を追加します。これにより、普段は完全に投資を忘れて本業に没頭できます。
NISAやiDeCo等のコア資産との棲み分け明記:
全世界株式などのインデックス投資を「コア（放置）」とし、このAIBAで見つけた次世代技術テーマを「サテライト（長期の成長枠）」としてトッピングする、というポートフォリオの立ち位置をサイト内で明確にすると、ユーザー側も「どう放置すればいいか」の運用イメージが湧きやすくなります。
