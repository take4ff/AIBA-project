**Advanced Investment & Behavior Analytics**

## 🎯 監視ターゲットの3層構造（タイムライン別）

時間軸に合わせて領域を3つに分類し、それぞれのフェーズに合った指標で市場を監視する。

| 階層（フェーズ） | 領域の具体例 | 投資の狙い・特徴 |
|---|---|---|
| **第1層：現在のブーム**<br>*(Current Trend)* | 先端半導体（GPU）、生成AI基盤、クラウドインフラ | 既に過熱しているため RSI 等のテクニカル指標を重視し、**「過熱感が収まった押し目」**を的確に狙う。 |
| **第2層：次なる波**<br>*(Next Wave)* | 冷却・電力インフラ、エッジAI・ロボティクス、バイオインフォマティクス | 今後1〜3年で実需が急拡大する領域。テクニカルとセンチメントの**「バランスの取れた成長」**を捉える。 |
| **第3層：未来の開拓地**<br>*(Future Frontier)* | 量子コンピューティング、宇宙インフラ・通信網 | 5〜10年先を見据えた領域。株価が動く前の、GitHub や学術界での**「水面下の研究開発の熱量」**を先行指標として拾う。 |

監視対象の具体的なティッカー・キーワードは [`config/targets.yaml`](./config/targets.yaml) で定義。

## 🧮 コア・アルゴリズム（AIBAスコア v1.1）

領域の階層に応じて指標の重み付けを動的に変化させるスコアリングモデル（0〜100）。実装は [`backend/aiba/score.py`](./backend/aiba/score.py)。

**インプットデータ**
- **テクニカル（事実）**：株価・出来高・RSI(14日)・移動平均(25日)乖離率（取得元 `yfinance`）
- **センチメント（先行）**：GitHub リポジトリの増加率・arXiv 論文の出現増加率（取得元 `GitHub API` / `arXiv API`）

**判定ロジック**
- **テクニカル割安感** = RSI割安スコア（低RSIほど高得点）と移動平均乖離スコア（平均より下ほど高得点）の平均。
- **層別重み** = 第1層 (テクニカル0.7 / センチメント0.3)・第2層 (0.5 / 0.5)・第3層 (0.3 / 0.7)。
- **RSIペナルティ** = RSIが50を超えた分を減点し、過熱状態の高値掴みを回避。
- **最終スコア** = 重み付き合算 − RSIペナルティ（0〜100にクランプ）。

## 🏗️ システムアーキテクチャ

日々の運用コストをゼロ（無料枠）に抑えつつ、自動化と保守性を高めた構成。

```
yfinance / GitHub API / arXiv API
            │  (収集)
            ▼
   Python バックエンド  ──[GitHub Actions: 日次cron]
   (pandas / requests)
            │  (スコア計算済みの日次サマリーのみInsert)
            ▼
   Supabase (PostgreSQL Free)
            │  (publishableキーで読み取り / RLS保護)
            ▼
   Next.js ダッシュボード (Vercel)
```

- **バックエンド**：Python（`pandas`, `requests`, `yfinance`）— [`backend/`](./backend/)
- **バッチ自動化**：GitHub Actions（cron、米国市場の閉場後に日次実行）— [`.github/workflows/daily.yml`](./.github/workflows/daily.yml)
- **データベース**：Supabase（PostgreSQL Free）。無料枠を活かすため計算済みの日次サマリーのみ保持 — [`db/schema.sql`](./db/schema.sql)
- **フロントエンド**：Next.js（App Router）。階層別ランキング（ヒートマップ）と時系列チャート — [`frontend/`](./frontend/)

### ディレクトリ構成
```
AIBA-project/
├── config/targets.yaml         # 監視対象（3層×ティッカー＋キーワード）
├── db/schema.sql               # Supabase スキーマ（テーブル/ビュー/RLS）
├── backend/                    # Phase 1: 収集・スコア計算・DB投入
│   ├── aiba/{config,technical,sentiment,score,db,pipeline}.py
│   ├── run_daily.py            #   日次バッチのエントリポイント
│   └── backfill.py             #   過去数ヶ月分のバックフィル
├── frontend/                   # Phase 3: Next.js ダッシュボード
└── .github/workflows/daily.yml # Phase 1: 日次自動実行
```
