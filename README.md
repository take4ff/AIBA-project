# 📊 次世代技術投資分析システム「AIBA（Advanced Investment & Behavior Analytics）」開発計画書

## 1. プロジェクトの目的

世間の熱狂による「割高な高値掴み」を避け、水面下で熱を帯びている次世代技術領域の最適な投資タイミング（押し目・初動）を定量的に検知する。テクニカル指標とセンチメント指標の乖離を分析し、データドリブンな投資決定を支援するダッシュボードシステム「AIBA」を構築する。

## 2. 監視ターゲットの3層構造（タイムライン別）

時間軸に合わせて領域を3つに分類し、それぞれのフェーズに合わせた指標で市場を監視します。

| 階層（フェーズ） | 領域の具体例 | 投資の狙い・特徴 |
| --- | --- | --- |
| **第1層：現在のブーム**<br>

<br>*(Current Trend)* | 先端半導体（GPU）、生成AI基盤、クラウドインフラ | 既に過熱しているため、RSI等のテクニカル指標を重視し、**「過熱感が収まった押し目」**を的確に狙う。 |
| **第2層：次なる波**<br>

<br>*(Next Wave)* | 冷却・電力インフラ、エッジAI・ロボティクス、バイオインフォマティクス | 今後1〜3年で実需が急拡大する領域。テクニカルとセンチメントの**「バランスの取れた成長」**を捉える。 |
| **第3層：未来の開拓地**<br>

<br>*(Future Frontier)* | 量子コンピューティング、宇宙インフラ・通信網 | 5〜10年先を見据えた領域。株価が動く前の、GitHubや学術界での**「水面下での研究開発の熱量」**を先行指標として拾う。 |

## 3. コア・アルゴリズム（AIBAスコア v1.1）

領域の階層（フェーズ）に応じて、指標の重み付けを動的に変化させるスコアリングモデルを採用します。

* **インプットデータ:**
* **テクニカル（事実）:** 株価、出来高、RSI（14日間）、移動平均乖離率（取得元：`yfinance`）
* **センチメント（先行）:** GitHubのリポジトリ増減、関連論文のキーワード出現率（取得元：`GitHub API`, `arXiv API`等）


* **判定ロジック（基本方針）:**
* RSIが50以上の過熱状態はスコアにペナルティを課す。
* 第1層（現在）はテクニカル指標の「割安感」への配点を高く設定。
* 第3層（未来）はセンチメント指標の「熱量の増加率」への配点を高く設定。



## 4. システムアーキテクチャ

日々の運用コストをゼロ（無料枠）に抑えつつ、自動化と保守性を高めた構成です。

* **データ収集・処理（バックエンド）:** Python (`pandas`, `requests`, `yfinance`)
* **バッチ処理・自動化:** GitHub Actions
* `cron`機能を利用し、米国市場の閉場後（日本時間早朝）に日次でスクリプトを自動実行。


* **データベース:** Supabase (PostgreSQL - Free Plan)
* 無料枠（500MB）を最大限活かすため、計算済みの「日次サマリー（スコア、終値、RSI等）」のみをInsertする設計。


* **可視化（フロントエンド）:** Next.js
* Supabaseからデータをフェッチし、階層ごとのランキングや時系列チャートをUIとして表示。


* **開発支援ツール:** Cursor, Claude Code

## 5. 開発ロードマップ

### Phase 1: データ基盤の構築と自動化（バックエンド）

1. 各領域に対応するETFティッカーと、センチメント測定用キーワードの確定。
2. Supabaseプロジェクトの立ち上げと、時系列データを格納するテーブル設計（DDL作成）。
3. Pythonによるデータ収集・スコア計算・SupabaseへのInsert処理の実装。
4. GitHub Actionsのワークフロー（`.yml`）を設定し、日次実行の稼働テスト。

### Phase 2: データ蓄積とアルゴリズムのキャリブレーション

1. GitHub Actionsによる自動運用を開始し、本番データを蓄積。
2. 蓄積されたデータと実際の市場の動き（ニュースや値動き）を比較。
3. スコア計算式（重み付けや閾値）を実態に合わせて微調整（キャリブレーション）。

### Phase 3: ダッシュボードの実装（フロントエンド）

1. Next.jsプロジェクトのセットアップ。
2. Supabase APIを用いたデータフェッチ処理の実装。
3. 3つの階層（現在・中期・未来）を切り替え、視覚的に「買い時」を判断できるヒートマップやチャートUIの構築。
