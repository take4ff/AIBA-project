# 📊 AIBA — 次世代技術投資分析システム

**Advanced Investment & Behavior Analytics**

世間の熱狂による「割高な高値掴み」を避け、水面下で熱を帯びている次世代技術領域の最適な投資タイミング（押し目・初動）を定量的に検知する。テクニカル指標とセンチメント指標の**乖離**を分析し、データドリブンな投資判断を支援するダッシュボード。

- **本番稼働中**：GitHub Actions が平日日次でデータ収集・スコア計算・予測 → Supabase 蓄積 → Vercel で公開。
- セットアップ・運用手順は **[SETUP.md](./SETUP.md)** を参照。

## ✨ 主な機能

### 買い（発掘）
- **AIBAスコア v1.2**：層別重み付け＋RSIペナルティの総合「買い時度」(0–100)
- **成長×割安スコア**／**センチメント傾き(↑↓)**／**🔀乖離**（センチメント先行×株価出遅れ＝仕込み好機）
- **保有目安バッジ（買いの"型"）**：🌱成長型（構造的成長・長期向き）／⚡反発型（売られすぎリバウンド・短期向き）／⚖️両取り
- **ピア上昇中バッジ**：同テーマ内ピア平均AIBAと比べ10点以上出遅れている銘柄を自動検出（IPO直後・低流動性銘柄の初動把握に有効）
- **1ヶ月先予測**：買い場入り確率＋予測AIBA（平均回帰＋ロジスティック。`predict.py --backtest` で naive 比較検証済み）
- **購入目安**：妥当値(25日MA)・押し目買い目安(−1σ)・下値支持(60日安値)を銘柄詳細に表示
- **下方リスク・下値メド**：52週安値（下落余地%）＋近い支持(−2σ/60日安値)、年率ボラ・1年最大下落率から**値動きの安定度（下方耐性）**を提示
- **⭐ Pickupページ**：地域・ETF/個別を問わず「今買い」候補を横断抽出（**前日順位との変動 ▲▼/NEW** を併記）
- **🔎 スクリーナー** `/screener`：地域/種別/階層/AIBA/買い場確率/予想PER/熱量/乖離の多条件を即時フィルタ＋並び替え。**条件は localStorage に自動保存**

### 売り（保有管理）
- **マイ・ポートフォリオ**（ログイン・アカウント紐付け）：保有銘柄を追加/編集/削除。**投信**は同じ指数を追う代用ETFで評価
- **過熱度**＋**売り場ハイライト**＋**売りシグナル合成**（テクニカル過熱＋ファンダ〈高PER/減益〉＋決算接近）
- **損切りライン**（任意・%可変）：取得単価から設定％下落で「損切り検討」
- **保有期間別の判定**（短期〜1ヶ月／中期1〜6ヶ月／長期半年〜）：期間ごとに適した指標群で「売り / 継続保有」を集計表示
- 平均取得単価比の**損益％**、次回決算日アラート
- **配分分析**：保有のテーマ別/地域別エクスポージャー＋集中度（最大比率・実効銘柄数＝1/HHI）
- **今後の決算カレンダー**：保有銘柄の決算予定日を日付昇順でカード表示（7日以内=赤・30日以内=橙）

### 銘柄詳細
- **一行ナラティブ**（自動要約）：AIBA＋相対PER＋業績＋熱量＋予測＋決算接近を1〜2文に
- **健康度レーダー**：AIBA/割安/熱量/非過熱/押し目度/**事業の頑丈さ**＋**業界平均を重ねて比較**
- **事業の頑丈さ（0–100）**：営業利益率・ROE・D/E・流動比率・FCF の合成スコア
- **相対PERフェアバリュー**：同地域・同テーマのピア中央値比で「◯%割安/割高」
- **時系列チャート**：株価×スコア二軸、買い場/売り場ハイライト、ボリンジャーバンド、MACD、通貨対応、期間切替
- **スイングトレンドライン**：直近2つの高値（レジスタンス・赤）と安値（サポート・緑）を自動検出し右端まで延長表示
- **テクニカル総合判定**：移動平均(25/200)・MACD・一目均衡表＋RSI・ボリンジャー・ストキャスを買い/売り/中立で網羅

## 🧮 AIBAスコア v1.2

階層に応じて指標の重みを動的に変える 0–100 のスコア（[`backend/aiba/score.py`](./backend/aiba/score.py)）。

**インプット**
- **テクニカル（事実）**：株価・出来高・RSI(14)・移動平均(25)乖離率（`yfinance`）
- **センチメント（先行）**：GitHub（新規リポジトリ＋コミット活動量）× arXiv（論文）× Hacker News × Google Trends × 特許（EPO OPS）× ニュース報道量（GDELT）× マクロ需要（FRED）の加重平均増加率
  - 本体=GitHub/arXiv（weight 1.0）、補助=HN/Trends/特許/ニュース/FRED（weight 0.4）。最低ボリューム・信号数ガードあり
  - GDELT用キーワードは研究語（arXiv）とは別に `gdelt_keywords` で市場向け語句を設定可能
  - FRED `IPG3344S`（半導体生産指数）は `advanced_semiconductor` / `ai_memory_storage` のみ有効（`FRED_API_KEY` 必要）

**ロジック**
- テクニカル割安感＝RSI割安スコアと移動平均乖離スコアの平均
- 層別重み＝第1層 (T0.4/S0.6)・第2層 (0.5/0.5)・第3層 (0.3/0.7)
- RSIが50超で過熱ペナルティ（MAの傾きで動的に閾値を緩和）→ 重み付き合算 − ペナルティ（0–100にクランプ）

## 🏗️ アーキテクチャ

運用コストを無料枠に抑えた構成。**重い処理は GitHub Actions、Vercel は表示のみ**。  
表示は **ISR（10分キャッシュ）＋ 重い集計の `unstable_cache`** で高速化。長いランキングは**初期表示＋「もっと見る」**で初期描画を軽量化。

```
yfinance / GitHub / arXiv / Hacker News / Google Trends / EPO OPS（特許）/ GDELT（ニュース）/ FRED（マクロ）
        │ 収集・スコア計算・予測（Python）
        ▼  ── GitHub Actions（日次cron＋週次cron）
Supabase (PostgreSQL Free)
        │ 公開データは anon キーで読み取り / RLS 保護
        │ ユーザーデータ(ウォッチリスト/保有)は本人のみ
        ▼
Next.js ダッシュボード (Vercel)
```

## 📈 キャリブレーション記録

out-of-sample で定点観測し、重み調整の判断材料を残す（`backend/backtest.py` / `snapshot.py`、H=21営業日基準）。詳細は `/verify`（検証）で常時確認可能。

### 主な結論（2026-06・2022〜2026で再評価、実センチメント全期間）
- **センチメントが選別力の源泉**：クロスセクションICで、センチメント **+0.021（IR +0.18・最も安定して正）**。一方 **AIBA −0.006 / テクニカル −0.021 と僅かに負**＝この**モメンタム主導相場では「割安・売られすぎ」の逆張り成分が相対的に劣後**したため。
  - ※ICの絶対値が小さいのは正常（市場は概ね効率的。単一factorは0.02〜0.05でも有意）。テクニカルの負ICは**地合い依存**（調整・レンジ相場では正に転じ得る）。
- **AIBA≥60 の選別力**：対象 22,510件 / 平均先行 **+4.01%**（全体 +2.84%）。
- **層別最適重み**：第1層=w(テク)0.7 / 第2層=0.1 / 第3層=0.0。2022年（弱気相場）のデータが加わったことで第1層のテクニカル有効性が復活している。
- **疑似エクイティカーブ**（月次リバランス・100→・2022〜）：**≥60** が分散の効いた頑健な優位（ユニバース等ウェイト・ACWI放置を上回る）。

**先行性の現状評価**：センチメントの正のIC＋IRはこの仮説を**支持する兆し**だが、(1)生存者バイアス＋in-sample、(2)ほぼ上昇・下落2レジームのみ のため**"有望な仮説"段階＝実証はこれから**。ICは `/verify` に時系列グラフ化し、時間経過で先行性が持続するかを評価できるようにしている。

**重要な注意（過大評価バイアス）**：ユニバースは"今の勝ち組"を後知恵で選定（生存者バイアス）、月次アンカーは**先行期間が重複**。絶対リターンでなく**相対優位・頑健性**として読むこと（取引コスト未考慮の概算）。

> 2026-06 保守メモ：①センチメント統合の**単一ソース/低ボリューム極値バグ**を修正（加重ブレンド＋ガード導入）、②**GDELT＋FRED**を補助信号に追加、③**全期間バックフィル**（2022〜）で GitHub/arXiv/HN 等の実センチメントを取得済み（flat 50 は新テーマの活動量ゼロ期間のみ・全体の2%未満）。

### v1.2 重み改定（2026-06）
**データ監査**：初期バックフィルは `--no-sentiment` で実行されていたため2022-2024が flat 50 だったが、2026-06 に `--sentiment-every-days 30` で再バックフィルを実施。**2022年以降の全データに実センチメントが入った状態で再評価済み**。

**再計算（2025+のクロスセクションIC）**：第1層は **純センチメントIC +0.146（システム最強）／純テクニカル 負**。現行 v1.1 は **テクニカル0.7** と逆だった。

**改定**：第1層を **テク0.7/センチ0.3 → 0.4/0.6**（センチメント寄りの中間）。第2/3層は据置。**過去スコアは v1.1 のまま**（定点記録の point-in-time 性を保持）。

## 🔬 派生スコアのロジック

銘柄詳細・ポートフォリオ等で使う派生スコアの定義・しきい値・近似/前提。`/guide`（利用者向け説明）に対し、ここは**式レベルの仕様**。

| スコア（実装） | 定義（要点） | 主なしきい値 | 近似/前提 |
|---|---|---|---|
| **事業の頑丈さ** 0–100<br>`fundamentals.ts:qualityScore` | 収益性(営業利益率・ROE)／財務健全性(D/E・流動比率)／キャッシュ創出(FCF符号) の利用可能指標の平均 | 営業利益率 20%≈70点/40%で満点・赤字で30以下／ROE 15%≈76点／D/E 100%≈55点・222%で0／流動比率 2.0≈90点 | D/E は yfinance の%表記前提。指標2つ未満は非表示 |
| **下方リスク・下値メド**<br>`indicators.ts:downsideProfile` | 下値メド=52週安値／近い支持=max(60日安値, −2σ)／安定度=年率ボラ＆1年最大ドローダウン | 安定度: ボラ<35%&DD>−35%→高 / <60%&>−55%→中 / 他→低 | テクニカルな目安で割れることあり |
| **保有期間別 売り/継続**<br>`indicators.ts:holdingHorizons` | 期間別に適した指標群の売り票 vs 継続票で判定 | 短期=過熱/RSI/ストキャス/BB/MACD・中期=MACD/一目/MA25,75・長期=MA200/52週/雲/熱量 | 票が拮抗は中立 |
| **テクニカル総合判定**<br>`indicators.ts:technicalSummary` | トレンド系(MA25/200・MACD・一目)＋オシレーター系(RSI・BB・ストキャス)を 買い/売り/中立 で集計 | 総合: 買い−売り の差で買い寄り〜売り寄り | 一目・ストキャスは終値ベース近似 |
| **未来GAFAM**<br>`platform.ts:platformScore` | 成長0.25／展開幅0.20／伸びしろ0.20／熱量0.20／頑丈さ0.15 の加重 | 伸びしろ＝時価総額が小さいほど高（対数: $5B≈満点, $1T≈12） | 投機的ヒューリスティック・将来非保証 |
| **損切り/利確**<br>`sell-signal.ts` | 取得単価比 −X% / +X% で検討バッジ | 既定 −20% / +30%（可変・端末保存） | 機械的判断の目安 |
| **AIBA指数(3型)**<br>`AibaIndexView.tsx` | ≥60全銘柄／上位20／各テーマ最上位 を月次等ウェイトで連結 | snapshot の ret_1m を複利連結・対ACWI | in-sample・取引コスト未考慮 |

**監査結論（2026-06）**：データは全件最新で整合（停止/孤児/重複なし）、スコア分布も直感と一致（例: 事業の頑丈さ NVDA 96／CoreWeave 14）。式レベルのバグは未検出。

| **銘柄** | キオクシアホールディングス（285A.T）・NANDフラッシュ最大手の一角 |
| **IPO** | 2024年12月18日、公募価格 ¥1,455（初値は僅かに公募割れ） |
| **上昇幅** | 2025年に **+540%**、2026年6月時点で公募比 **約70倍**（¥10万超）|
| **主因** | AI推論（inference）向けエンタープライズSSD需要爆発 → 2026年生産枠完売 → NAND価格高騰 |

### AIBAは捉えられていたか？

**◎ 既に対応していたこと**
- 285A.T は `advanced_semiconductor` JP銘柄に上場前から定義済み → IPO翌日から追跡開始

**✗ 捉えられていなかったシグナル**

AIが「AIはメモリで動く」時代に入ったことを示す研究・開発活動（GitHub/arXiv）は2024年前半から急増していたが、`advanced_semiconductor` のキーワードが **compute 層（cuda / gpu kernel / tensorrt）に偏っており memory/storage 層を捉えていなかった**。

| 見逃した先行シグナル | 発生時期 |
|---|---|
| vLLM・LMCache などKVキャッシュ管理フレームワークのGitHubスター急増 | 2024 Q1–Q2 |
| arXiv: KV cache offloading to NVMe SSD 論文の増加 | 2024 Q2–Q3 |
| NANDコントラクト価格の上昇転換 | 2024 Q4 |
| キオクシア IPO | 2024-12 |
| 株価 +540% | 2025 通年 |

### 実施した改善

1. **`advanced_semiconductor` キーワード強化**：GitHub/arXiv にメモリ・推論層のキーワードを追加（`kv cache`, `nvme ssd`, `memory bandwidth`, `flash attention` 等）

2. **`generative_ai` キーワード強化**：推論フレームワーク活動を捉えるため `vllm`, `inference engine`, `speculative decoding` 等を追加

3. **新テーマ `ai_memory_storage`（第2層）を追加**：銘柄 MU/WDC/285A.T/000660.KS、専用ETF DRAM（Roundhill Memory ETF）を us に配置、`gdelt_keywords: "NAND price"` で市場ニュースも捕捉

4. **5テーマのキーワードを追加強化**：cloud_infra / cooling_power_infra / edge_ai_robotics / bioinformatics / cybersecurity

5. **GDELTキーワードをarXivから分離**（`gdelt_keywords` フィールド追加）：9テーマで研究語と市場語を分離し、ニュース報道量取得の精度を向上

6. **ピアモメンタム（`peer_avg_aiba`）実装**：同テーマ内ピア平均との乖離を検出し「ピア上昇中」バッジを表示。IPO直後や低流動性銘柄の初動把握に対処

7. **FRED 半導体生産指数（`IPG3344S`）を補助センチメント信号に統合**：直近3ヶ月 vs 前3ヶ月の成長率を 0-100 スコアに変換（WEIGHT_SUPP=0.4）。`advanced_semiconductor` / `ai_memory_storage` のみ有効

8. **専用メモリETF（DRAM）を追加**：`ai_memory_storage` の us ETF を SMH（広域半導体）から Roundhill Memory ETF（DRAM）に変更。global ETF は SMH を維持してベンチマーク用途に残す

### シグナルカバレッジの評価

**課題①：IPOタイミング（構造問題）**  
上場前の個別銘柄追跡は不可能。上場直後はテクニカル指標が基準値を持てずAIBAスコアが安定するまで数ヶ月かかる。「ピア上昇中」バッジが部分的に補完するが根本解決ではない。

**課題②：シグナル種別のミスマッチ**  
AIBAのセンチメントは「研究熱量→株価」の先行性を前提とするが、Kioxiaは「需要ショック→NAND価格→株価」というコモディティサイクル構造だった。

| 先行シグナル | カバー状況 |
|---|---|
| NANDスポット・コントラクト価格 | ❌ 有料データ・対応困難 |
| ハイパースケーラー設備投資発表 | ✅ `gdelt_keywords: "NAND price"` で部分カバー |
| GitHub/arXiv のメモリ研究活動増加 | ✅ キーワード強化で対応済み |
| ピア銘柄（MU/WDC）の先行上昇 | ✅ `ai_memory_storage` テーマ＋「ピア上昇中」バッジ |
| 半導体生産サイクルの方向感 | ✅ FRED `IPG3344S` 補助信号で統合済み |
| 専用メモリETFでの業界トレンド把握 | ✅ Roundhill Memory ETF（DRAM）を追加 |

### 🔬 スコア改善の仮説（未検証）

**仮説①：CoreWeave / GPU-as-a-Service 問題（ケーススタディ候補）**

CoreWeave（CRWV）は2025年3月IPO後に急騰したGPU専用クラウド企業。現在 `cloud_infra` に含まれているが、AWS/Azure と同じテーマに入れると研究キーワードの文脈が異なり埋もれる可能性がある。キオクシア調査と同構造の「テーマのキーワードミスマッチ」がないか、`cloud_infra_us_crwv` のセンチメント履歴とIPO前後の株価を照合して確認する価値がある。

**仮説②：センチメント「水準」vs「変化率」問題**

現在のセンチメントスコアは増加率（モメンタム）のみを測定しているため、熱量が高水準で横ばいになると sentiment_score が下がりAIBAも下がる設計になっている。「熱量が増え続ける」フェーズ（急拡大期）と「熱量が高原を維持する」フェーズ（成熟期）では別の評価が必要かもしれない。

```
改善案: sentiment_score = α × 増加率スコア + (1-α) × 水準スコア
期待: 先端半導体のような成熟した熱狂期の銘柄を過小評価しない
```

**仮説③：GDELTトーン変化を先行指標に（実装コスト低）**

現在GDELTは「報道量（Volume）」のみをスコア化しているが、GDELT APIは `AvgTone`（平均論調: −100〜+100）も返している。週次ジョブ（`news_tone_job`）ではテーマ別トーンを集計済みだが、daily_metricsには反映していない。報道量とトーン変化の両方を補助信号にすると、同じ報道量でもネガティブ化・ポジティブ化を区別できる。

```
実装イメージ: gdelt_tone_score = f(AvgTone の直近変化率) → WEIGHT_SUPP で加算
```

**仮説④：RSIペナルティの「モメンタム強度」補正**

NVDAのように `sentiment_trend = ↑` かつ株価が上昇トレンドを継続する銘柄は、RSI=80超が常態化してもAIBAが50台に留まる。ただしキャリブレーション記録でテクニカルスコアのICが僅かに負だったことから、RSIペナルティ補正より先に「センチメント一本化（w_tech=0への移行）」の効果を実測する方が先決かもしれない。

**仮説⑤：日本株のROE改善「速度」**

東証の2023〜コーポレートガバナンス改革でPBR1倍割れ是正が進行中。現在「事業の頑丈さ」のROEは時点の水準（15%≈76点）のみを評価しているが、ROEの改善トレンド（前年比増分）を加えると東証改革の恩恵銘柄（改善モードに入った日本株）を正しく評価できる可能性がある。
